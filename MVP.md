# Bed Game MVP 文档

## 1. 目标
- 在不做账号系统的前提下，快速交付一个可玩的情侣亲密任务抽卡网页。
- 支持双人/单人两种模式，具备随机任务、计时、回合推进、结束与 aftercare。
- 强化“自愿、可随时停止、边界优先”的交互规则。

## 2. MVP 范围（必须做）
- `入口页`：项目介绍 + 18+ 年龄确认 + 开始按钮。
- `流程页（多步）`：
  - 第一步：双人游玩 / 单人游玩。
  - 第二步（仅双人）：强度档位（清水/亲密/调皮/随机）。
  - 第三步（仅双人）：回合方式（轮流 or 选择谁先开始）。
  - 第四步：昵称输入（提供默认名，可改）。
  - 交互要求：一个选项一个页面，点选后自动滑到下一页。
- `规则确认页`：
  - 明示双方自愿、可跳过、可随时叫停。
  - 勾选“我已同意规则”后才能进入游戏。
- `游戏页`：
  - 显示 3 张任务卡，玩家每轮点选并抽取 1 张。
  - 抽取后不在当前页直接执行任务，而是弹出独立任务窗口。
  - 任务完成后关闭弹窗，并自动刷新下一轮 3 张新卡。
  - 任务进行中禁用再次抽卡。
- `任务弹窗`：
  - 任务名称。
  - 开始任务按钮。
  - 当前任务计时（按下开始任务后计时）。
  - 停止任务按钮（停止后开始下一次抽卡）。
  - 跳过任务（每人每局 1 次，可选放在弹窗次级操作）。
- `结束页`：
  - 主动结束游戏按钮始终可见。
  - 结束后进入 aftercare（双人/单人分支文案）。

## 3. 明确不做（MVP 外）
- 登录注册、云端存档、历史记录同步。
- 排行榜、社交分享、房间联机。
- 复杂动画系统与音效编辑器。
- 后台运营系统（任务在线配置）和 A/B 实验平台。

## 4. 任务与数据最小模型
- 建议任务 JSON 字段：
  - `id`
  - `title`
  - `mode` (`solo` | `duo`)
  - `intensity` (`light` | `intimate` | `playful`)
  - `durationSec`
  - `tags`（如 `warmup`, `consent-friendly`）
  - `cooldown`（避免连续重复）
- 运行态最小状态：
  - `currentPlayer`
  - `roundIndex`
  - `drawLocked`
  - `currentDrawCards`
  - `taskModalOpen`
  - `taskTimerStatus` (`idle` | `running` | `stopped`)
  - `skipRemainByPlayer`
  - `selectedMode` / `selectedIntensity`

## 5. 关键规则（MVP 必须落地）
- 开局边界确认：每位玩家至少选择一次“可接受/犹豫/不可接受”范围。
- 过滤逻辑：命中“不可接受”标签的任务不进入抽取池。
- 随机模式：从三档按权重抽取（建议 `0.4/0.4/0.2`）。
- 去重逻辑：最近 2 次命中过的任务本轮不再出现。
- 轮换逻辑：双人默认严格轮流；若选择“指定先手”，第一回合按指定，后续轮流。

## 6. 验收标准（Done 定义）
- 用户可在 2 分钟内完成配置并进入第一轮任务。
- 双人模式可连续完成至少 5 轮，无卡死、无状态错乱。
- 任务弹窗打开时无法重复触发抽卡。
- 点击“开始任务”后计时器开始；点击“停止任务”后关闭弹窗并刷新三张卡。
- 跳过次数、轮流顺序、计时器显示准确。
- 结束游戏后一定进入 aftercare 页面。
- 刷新页面后不报错（是否保留进度可选，但必须可重新开始）。

## 7. 实现建议（最小技术方案）
- 前端单页：`HTML + CSS + Vanilla JS`（先快跑）。
- 数据：本地 `tasks.json` + 内存状态管理。
- 持久化：`localStorage`（仅保存最近一次配置，可选）。
- 部署：静态托管（Vercel/Netlify/GitHub Pages 均可）。

## 8. 里程碑（建议）
- `M1`：完成流程页与规则确认页（可走通）。
- `M2`：完成三卡抽取机制 + 任务弹窗 + 计时器。
- `M3`：完成单双人回合逻辑 + 跳过 + 去重。
- `M4`：完成 aftercare + 文案润色 + 基本测试。
